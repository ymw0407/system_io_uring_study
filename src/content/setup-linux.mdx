<Section title="Linux 셋업 — QEMU + KVM" id="setup-linux">

이 페이지는 호스트가 **Linux**일 때 실습 환경을 처음부터 만드는 방법을 다룬다.
다른 OS는 [Setup 인덱스](/setup)로 돌아가서 선택한다.
KVM은 보통 Linux 커널에 이미 포함돼 있어 추가 모듈 설치가 거의 없고, 가속을 켜는 데 필요한 권한만 확인하면 된다.

## 1. QEMU 설치

배포판 패키지로 설치한다.

<Terminal title="Ubuntu / Debian" commands={[
  { cmd: "sudo apt update" },
  { cmd: "sudo apt install -y qemu-system-x86 qemu-utils" }
]} />

<Terminal title="Fedora" commands={[
  { cmd: "sudo dnf install -y qemu-system-x86 qemu-img" }
]} />

<Terminal title="Arch" commands={[
  { cmd: "sudo pacman -S qemu-full" }
]} />

설치 후 버전 확인.

<Terminal title="QEMU 버전" commands={[
  { cmd: "qemu-system-x86_64 --version", output: ["QEMU emulator version 8.2.0"] }
]} />

## 2. KVM 가용성 확인

KVM이 활성화되어 있고 권한이 있는지 확인한다.

<Terminal title="KVM 가용성" commands={[
  { cmd: "ls /dev/kvm && egrep -c '(vmx|svm)' /proc/cpuinfo", output: ["/dev/kvm", "8"] }
]} />

`/dev/kvm`이 존재하고 CPU 플래그(vmx 또는 svm)가 1 이상이면 KVM 가속이 가능하다.
`/dev/kvm` 권한이 없다는 에러가 나면 다음을 실행하고 재로그인한다.

<Terminal title="KVM 그룹 가입" commands={[
  { cmd: "sudo usermod -aG kvm $USER" }
]} />

## 3. Ubuntu ISO 다운로드

[releases.ubuntu.com/22.04](https://releases.ubuntu.com/22.04/)에서 **Ubuntu 22.04 LTS Server**의 ISO를 받는다.
파일명은 보통 `ubuntu-22.04.x-live-server-amd64.iso` 형태다.
받은 파일을 실습 디렉터리에 두고, 다음 단계의 명령은 모두 그 디렉터리에서 실행한다.

<Terminal title="실습 디렉터리 만들기" commands={[
  { cmd: "mkdir ~/io-uring-lab && cd ~/io-uring-lab" }
]} />

`wget`이나 `curl`로 직접 받을 수도 있다.

<Terminal title="curl로 ISO 다운로드 (예시)" commands={[
  { cmd: "curl -LO https://releases.ubuntu.com/22.04/ubuntu-22.04.5-live-server-amd64.iso" }
]} />

## 4. 디스크 이미지 생성

OS용 디스크와 실험용 가상 NVMe 디스크를 분리해서 만든다.
나누는 이유는 NVMe 드라이버를 다시 로드하거나 디스크를 초기화할 때 OS를 건드리지 않기 위해서다.

<Terminal title="디스크 이미지 생성" commands={[
  { cmd: "qemu-img create -f qcow2 ubuntu.qcow2 30G" },
  { cmd: "qemu-img create -f raw nvme.raw 16G" }
]} />

`ubuntu.qcow2`는 OS용 30GB qcow2 이미지(실제 크기는 사용량만큼만 늘어남), `nvme.raw`는 실험 대상이 될 16GB raw 이미지다.

## 5. ISO로 부팅해서 Ubuntu 설치 (최초 1회)

ISO와 빈 디스크 두 개를 함께 부팅한다.

<CodeBlock language="bash" filename="install-vm-linux.sh">
{`#!/bin/bash
qemu-system-x86_64 \\
  -enable-kvm \\
  -m 4G -smp 4 \\
  -cdrom ubuntu-22.04.5-live-server-amd64.iso \\
  -drive file=ubuntu.qcow2,format=qcow2,if=virtio \\
  -drive file=nvme.raw,format=raw,if=none,id=nvme0 \\
  -device nvme,drive=nvme0,serial=test \\
  -netdev user,id=net0,hostfwd=tcp::2222-:22 \\
  -device virtio-net-pci,netdev=net0`}
</CodeBlock>

`-nographic`을 빼면 별도의 QEMU 창이 뜨고 거기서 마우스/키보드로 Ubuntu 설치 위저드를 진행할 수 있다.

설치 위저드에서 챙길 것:

- 언어·키보드 기본값으로 진행
- 네트워크: DHCP 자동 (포트 포워딩이 알아서 처리)
- 디스크: `ubuntu.qcow2` 전체를 LVM으로 사용 (NVMe 디스크는 건드리지 않음)
- 사용자명·비밀번호 설정
- **"Install OpenSSH server" 체크 필수** (이후 SSH 접속에 사용)

설치 완료 메시지가 뜨면 VM을 종료한다.

## 6. 설치된 VM 부팅

이후로는 ISO 없이 바로 부팅한다.

<CodeBlock language="bash" filename="run-vm-linux.sh">
{`#!/bin/bash
qemu-system-x86_64 \\
  -enable-kvm \\
  -m 4G -smp 4 \\
  -drive file=ubuntu.qcow2,format=qcow2,if=virtio \\
  -drive file=nvme.raw,format=raw,if=none,id=nvme0 \\
  -device nvme,drive=nvme0,serial=test \\
  -netdev user,id=net0,hostfwd=tcp::2222-:22 \\
  -device virtio-net-pci,netdev=net0 \\
  -nographic`}
</CodeBlock>

각 옵션의 역할:

- `-enable-kvm` — Linux의 KVM으로 하드웨어 가속
- `-m 4G -smp 4` — 게스트에 RAM 4GB, vCPU 4개
- `-drive ... if=virtio` — OS 디스크
- `-drive ... id=nvme0` + `-device nvme,...` — 가상 NVMe 디바이스 (실험 대상)
- `-netdev ... hostfwd=tcp::2222-:22` — 호스트 2222 → 게스트 22 SSH 포워딩
- `-nographic` — 별도 창 없이 현재 터미널에 콘솔 연결

부팅 로그가 끝나고 `login:` 프롬프트가 뜨면 거기서는 그대로 두고, 새 터미널에서 SSH로 접속하는 편이 편하다.

## 7. SSH 접속

새 터미널에서:

<Terminal title="SSH 접속" commands={[
  { cmd: "ssh -p 2222 ubuntu@localhost" }
]} />

설치 시 만든 사용자명·비밀번호로 접속한다.

## 8. 게스트 안에서 liburing 설치

이후 명령은 모두 **게스트(VM) 안의 셸에서** 실행한다.

<Terminal title="liburing과 빌드 도구 설치" commands={[
  { cmd: "sudo apt update" },
  { cmd: "sudo apt install -y liburing-dev gcc build-essential pkg-config" }
]} />

## 9. 환경 검증

### 9.1 커널 버전

<Terminal title="커널 버전" commands={[
  { cmd: "uname -r", output: ["5.15.0-91-generic"] }
]} />

5.1.x 이상이면 OK. Ubuntu 22.04는 기본 5.15(HWE).

### 9.2 liburing 버전

<Terminal title="liburing 버전" commands={[
  { cmd: "pkg-config --modversion liburing", output: ["2.1"] }
]} />

2.x가 출력되면 OK.

### 9.3 컴파일러 버전

<Terminal title="컴파일러 버전" commands={[
  { cmd: "gcc --version", output: ["gcc (Ubuntu 11.4.0-1ubuntu1~22.04) 11.4.0"] }
]} />

gcc 9 이상이면 OK.

### 9.4 NVMe 디바이스 인식

<Terminal title="가상 NVMe 디바이스 확인" commands={[
  { cmd: "lsblk -d -o NAME,SIZE,ROTA,TRAN | grep nvme", output: ["nvme0n1  16G  0  nvme"] }
]} />

`nvme0n1`이 보이고 `ROTA`가 0이면 가상 NVMe가 정상적으로 게스트에 노출된 상태다.

### 9.5 최소 io_uring 프로그램

<CodeBlock language="c" filename="check.c">
{`#include <liburing.h>
#include <stdio.h>
#include <string.h>

int main(void) {
    struct io_uring ring;
    int ret = io_uring_queue_init(8, &ring, 0);
    if (ret < 0) {
        fprintf(stderr, "io_uring_queue_init failed: %s\\n", strerror(-ret));
        return 1;
    }
    printf("io_uring 준비 완료: SQ entries = 8\\n");
    io_uring_queue_exit(&ring);
    return 0;
}`}
</CodeBlock>

<Terminal title="컴파일과 실행" commands={[
  { cmd: "gcc -O2 -o check check.c -luring" },
  { cmd: "./check", output: ["io_uring 준비 완료: SQ entries = 8"] }
]} />

이 출력이 나오면 Part 2 이후의 모든 코드를 돌릴 준비가 된 것이다.
